# 🎓 Student Success & Retention Analytics (SQL Case Study)

---

## 📌 Project Overview
This project focuses on analyzing student academic performance and engagement data using SQL.

The goal is to identify:
- At-risk students  
- Engagement patterns  
- Academic performance trends  
- Retention opportunities  

This project simulates a real-world academic analytics system used by institutions.

---

## 🎯 Objectives
- Perform **data quality checks** (duplicates, null values)
- Detect **outliers** in GPA and grades
- Create a custom **Engagement Score**
- Identify **high-performing but low-engagement students**
- Analyze **department-level performance**
- Build a **Retention Risk Dashboard**

---

## 🛠️ Tools & Technologies
- SQL (MySQL / PostgreSQL)
- CSV Dataset
- GitHub

---

## 📊 Dataset Description

| Column Name | Description |
|------------|------------|
| student_id | Unique student identifier |
| major | Student’s department |
| gpa | Grade Point Average (0–4 scale) |
| avg_course_grade | Average marks (0–100) |
| attendance_rate | Attendance percentage |
| lms_logins_past_month | LMS activity |
| assignment_submission_rate | Assignment completion rate |
| forum_participation_count | Forum activity |
| risk_level | Risk category (High/Medium/Low) |
| enrollment_status | Active / Leave |

---

## 📈 Key Insights

- Students with low engagement tend to score lower grades  
- High GPA + Low engagement = hidden risk group  
- Certain departments require academic support  
- Early intervention can improve retention
